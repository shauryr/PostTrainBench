## Version Control
You are working inside a git repository on an experiment branch. **Commit your work frequently** so progress is tracked:
- After writing a training script: \`git add -A && git commit -m "Add training script: <brief description>"\`
- After a training run completes: \`git add -A && git commit -m "Training complete: <results summary>"\`
- After evaluation: \`git add -A && git commit -m "Evaluation: <score>"\`
- Before starting a new approach: \`git add -A && git commit -m "Pivot: <reason>"\`
Do NOT commit the model weights (final_model/) or large binary files — only scripts, configs, and logs.
Add a .gitignore with: \`echo -e "final_model/\n*.pt\n*.bin\n*.safetensors\n__pycache__/" > .gitignore\` before your first commit.
